Day 3 In-class Exercise

info

This is an in-class exercise. An exercise page like this one will contain a brief description but is intended to be supplemented by discussion during our meeting time. Complete the exercise to the best of your ability. Feel free to talk with other students as you work, and do not be afraid to ask questions if you get stuck. Aim to complete as much as possible during our meeting, and try to finish on your own before our next class meeting. Remember to submit on Gradescope once or more, checking the autograder results. For credit, you must submit by the end of the semester.

Learning Objectives#

Objectives

This exercise will help you gain additional familiarity with:

  • Conditionals and loops
  • Arrays and strings
  • Characters and ASCII table
  • File I/O (fopen, fclose, fprintf, fscanf, ferror)
  • Math functions (pow & exp)
  • emacs, git, gcc, zip

Part 1 - Setup alias commands in bashrc#

So far, you have learned how to compile a C program from source files using this command:

gcc -std=c99 -pedantic -Wall -Wextra <source files>

It's a bit inconvenient to type this long command again and again for compiling different source files. To speed up, you can create alias commands.

  1. We created a .bashrc Unix shell file that contains several aliases and environment settings that will make your ugrad server lives better. Along with this, we have a .bash_profile configuration file that will get the .bashrc settings into place when you log in remotely. While you are still in the cs220-public directory, type ls -a to see this (hidden) file, along with the usual suspects.

  2. Assuming you do not already have a customized .bashrc file in your home directory (ask for help if you already have one!), type cp .bashrc ~ to copy our version of the file to your home directory. Next, type cp .bash_profile ~ to copy the other file to your home directory. Then type source ~/.bashrc to run it. Normally this configuration file will automatically run every time you log into a ugrad Unix server, but for now, we want it to take effect in this session.

  3. Type cat ~/.bashrc to see the contents of that file. It sets your default EDITOR to be emacs (feel free to change it to vim if you prefer, then re-type the source command above), and it defines two aliases we can use as shortcuts when compiling our programs. The first one, gccc (note the extra 'c'!) is our standard gcc command with all the required flags already included. To use it, just type gccc myfile.c to compile a file, which is equivalent to (an alias of) gcc -std=c99 -pedantic -Wall -Wextra myfile.c. The lines beginning with a hashtag are comments in this Unix shell scripting language. The second alias will be useful when we start using C++.

Checkpoints#

  1. What does gccc mean?
  2. Which one do you prefer to type, gccc <source files> or gcc -std=c99 -pedantic -Wall -Wextra <source files>?
  3. Try to define an alias of your own in ~/.bashrc. Have you created some useful alias commands that help speeding up your typesetting? e.g. Do you think this alias alias update_public='cd ~/cs220-public; git pull; cd ~' is helpful?

Part 2 - More C control flow (DNA reverse complement)#

Introduction#

In this part, you should work on count1.c. You will practice how to read a C string char by char and modify its value. Your goal is to apply the reverse complement on a DNA string. You should pay attention to the null-terminator of a C-string.

Instructions#

  1. Open the file for editing using emacs count1.c or vim count1.c.
  2. Read the source code and the comments. Your instructions are in the comments.
  3. Modify the source file and test by:
    • Compiling with `gccc count1.c -o count1 (use the new alias!) The extra -o count1 portion at the end of the line is asking for the executable created by gcc to be named count1 instead of the usual a.out.
    • Running with ./count1.
    • The results should be printed to the console for you to compare.

Checkpoints#

  1. Why does rev_comp have the size dna_len + 1, but not dna_len?
  2. Have you handled the null-terminator correctly?
  3. Which loop have you used? while loop, do while loop, for loop? Can you find a way to convert one to the other? How do you terminate your loop?
  4. Have you used the if-else statement or switch-case statement? Can you convert it from one to the other?

Part 3 - ASCII values (Counting characters in C-string)#

Introduction#

In this part, you will work on count2.c. You are given a C-string, and you want to count how many alphabetic, digit, and whitespace characters in it. While you can use isalpha, isdigit and isspace functions for the checking, you should challenge yourself to use the ASCII value for the comparison.

Instructions#

  1. Open the file for editing using emacs count2.c or vim count2.c.
  2. Read the source code and the comments. Your instructions are in the comments.
  3. Modify the source file and test by:
    • Compiling with `gccc count2.c -o count2.
    • Running with ./count2.
    • The results should be printed on the console for you to check.

Checkpoints#

  1. What are ASCII values?
  2. Is there a difference between char a = '1'; and char a = 1;?
  3. Can you convert an upper-case letter to a lower-case letter using the ASCII values?

Part 4 - More ASCII values (Most frequently-occurring characters)#

Introduction#

Now, you should work on count3.c. This time, you are asked to find the two most frequently occurring characters in a C-string. Because you now know that characters in C are represented by 256 ASCII values, you are going to count the frequency using an array of 256 integers and then use the counting result to find the two most frequently occurring characters.

Instructions#

  1. Open the file for editing using emacs count3.c or vim count3.c.
  2. Read the source code and the comments. Your instructions are in the comments.
  3. Modify the source file and test by:
    • Compiling with `gccc count3.c -o count3.
    • Running with ./count3.
    • The results should be printed on the console for you to check.

Checkpoints#

  1. Justify why you can use an array of 256 integers for the counting.
  2. How do you get the most frequently occurring character using the array?
  3. How do you get the second most frequently occurring character using the array?

Part 5 - File I/O and Math functions (Calculating compound interests)#

Introduction#

This is the last part of today's exercise. You are going to practice file I/O in C and use the math functions to implement different compound interest formulas. You should work on compound.c and use input.txt as your file input.

Instructions#

  1. Open the file for editing using emacs compound.c (or vim compound.c, if you use vim.)

  2. Read the source code and the comments. Your instructions are in the comments. You will find it handy to use this website for looking up formulas to solve the problem.

  3. Modify the source files and test by:

    • Compiling with gcc -Wall -Wextra -std=c99 -pedantic -lm compound.c -o compound or if you prefer to take advantage of the alias: gccc -lm compound.c -o compound.
    • Running with ./compound input.txt (change up to test with inputBad1.txt and inputBad2.txt also.)
    tip

    The gcc command we are using today includes the extra argument -lm which tells gcc to include the m (math) library when linking. Later this week, we will talk more about the relationship between header (.h) files, libraries, the compiler, and the linker.

    The interest rates in the input.txt file are expressed as fractions, not percents. So 0.3 means 30%, not 0.3%.

    When the compound program runs successfully, it places its output in a new file called output.txt. You will not see the output at the terminal. (You might see output at the terminal if your program fails, but you should not see any if it succeeds.) To check the contents of the file output.txt, use a command like cat output.txt or more output.txt. For your reference, in the instructors' solution, the first two lines of output.txt are:

    107.00 107.23 107.25
    580.25 582.74 582.98
    Unix Tip of the day

    We gave you the expected output file outputGood.txt, which is what your program should output when run with input.txt as input. Once you've run your code successfully, you can use the Unix diff command to see if your results in output.txt match the expected results in outputGood.txt as follows: diff output.txt outputGood.txt.

    No output from running this command means they are exactly the same. Otherwise, you will see lines with < or > at the start, indicating which lines differ, and how they appear in one file or the other. Another way to compare them is to open both in emacs - either in side by side terminal windows, or within emacs using the ctrl-x 4 f command (type 2nd filename when prompted). Use ctrl-x o (other) to toggle between the file buffers. Use ctrl-x 1 with the cursor in the window you want to keep when done looking at both.

Checkpoints#

  1. Why do we need -lm in our gcc/gccc command?
  2. How do you open a file to read in C?
  3. How do you open a file to write in C?
  4. How do you close a file in C?
  5. How do you check if there is a file error in C?
  6. How do you parse the file content and assign the values to variables in C?
  7. What is an assertion in C? When should you use it?
  8. What is the return value of fscanf?

Day 8 In-class Exercise

info

This is an in-class exercise. An exercise page like this one will contain a brief description but is intended to be supplemented by discussion during our meeting time. Complete the exercise to the best of your ability. Feel free to talk with other students as you work, and do not be afraid to ask questions if you get stuck. Aim to complete as much as possible during our meeting, and try to finish on your own before our next class meeting. Remember to submit on Gradescope once or more, checking the autograder results. For credit, you must submit by the end of the semester.

Learning Objectives#

Objectives

This exercise will help you gain additional familiarity with:

  • Linked lists
  • Dynamic memory allocation
  • Pointers, pointers, and more pointers
  • Recursion
  • Implementing length, add_after, and reverse_print functions
  • Implementing deletion from a (singly) linked list, and insertion into a sorted version of a linked list

Part 0 - Get the starter codes#

Get the starter codes from cs220-public, copy them over to cs220 (remember to create a new directory ex8 for that), and create an initial commit with the copied starter codes. You should see the following files: list.c, list.h, main.c, and a Makefile.

tip

If you don't remember how to get the starter codes, go back and check previous exercises for instructions.

Part 1 - Linked Lists#

Introduction#

In this exercise, you will implement the data structure linked list and the necessary functionality. The function prototypes are provided for you in list.h. Follow the instructions below to implement and test them one by one.

Instructions#

length#

  1. Implement the length function. The function is passed a pointer to the head node of a linked list, and returns an integer indicating how many values are stored in the linked list. (Equivalently, it returns a count of the number of nodes in the linked list, since each node stores one element.)

  2. Uncomment the assertion in main.c which tests length on a linked list.

add_after#

  1. Implement the add_after function. It takes a pointer to a Node and a char value as parameters and should insert a new node with the corresponding char value, after the node passed as the parameter.

  2. Uncomment the assertions and call to print in main.c to test add_after on a linked list.

print_rec#

  1. Implement the print_rec function. It is like the print_list function, except that it uses recursion to print the values. This one will require some (recursive) thought!

  2. Replace the call to print_list with print_rec in line 51 (in the add_after test code) of main.c to test your implementation.

reverse_print#

  1. Implement the reverse_print function. It is like the print_list function, except that it prints the values in reverse order. This one will require some (recursive) thought also!

  2. Uncomment the call to reverse_print in main.c to test your implementation.

clear_list#

  1. Implement the clear_list function. It releases all allocated memory in the list. It takes a pointer to the pointer variable that points to the head node of a linked list.

  2. Compile and run the program with valgrind to confirm there is no memory problem.

add_front#

  1. Implement the add_front function. Different from add_after, it takes a pointer to the pointer variable that points to the head node of a linked list, and should insert a new node with the input data to the front of the linked list.

  2. Uncomment the assertions and call to print in main.c to test add_front on a linked list.

remove_after and remove_front#

  1. Implement the remove_after and remove_front functions. The first removes and frees the node immediately after the argument. The second removes and frees the head of the list and updates the head pointer. Each one should return the character in the node being removed.

    For example, if your list includes the values a b c d e, then remove_after called on a pointer to the node containing c would result in a list containing a b c e, and would return d. And a call to remove_front on that revised list would result in a list containing b c e, and would return a.

  2. Uncomment the call to remove_after and remove_front in main.c to test your implementation.

  3. Run the program with valgrind to confirm there is no memory problem.

remove_all#

  1. Implement the remove_all function. This removes all nodes in the list whose value matches the argument.

  2. Uncomment the call to remove_all in main.c to test your implementation.

  3. Run the program with valgrind to confirm there is no memory problem.

insert#

  1. Implement the insert function. This function takes a pointer to the head of the list (which may point to NULL), creates a node with the prescribed value, and inserts it into the list in sorted order. This function will set the new head pointer if the head of the linked list has been updated.

    • You can assume that when the function is called, the linked list comes in sorted.
    • For sorting, use the < and > operators. (e.g. the character â€˜aâ€™ should be larger than the character 'B'.)
    • Feel free to make use of the add_front and add_after methods you have already written.

  2. Uncomment the call to insert in main.c to test your implementation.

  3. Run the program with valgrind to confirm there is no memory problem.

other functions#

  1. There are couple more functions declared in list.h. Check them out and implement them when you have time.

  2. After you have implemented those functions, uncomment the related part in main.c to test your implementation.

  3. Always run the program with valgrind to confirm there is no memory problem.

Checkpoints#

  1. What do we need to use pointer-to-pointer in clear_list, add_front and remove_front functions?
  2. When you implement the recursive print version, how do you figure out the recursive relationship?
  3. How can you extend your implementation to make it a doubly linked list?

Part 2 - Gradescope submission#

Create a zip file called ex8.zip, which contains list.c and list.h with your implementation (you don't need to submit main.c and Makefile). Submit the zip file as Day 8 In-class Exercise on Gradescope. Make sure your program passes all the autograder tests in order to get full credit. It is due before the next class. However, late submission is allowed until our last class. Even so, we highly recommend you to keep up the schedule, finish each in-class exercise, and submit it on Gradescope before the next class.

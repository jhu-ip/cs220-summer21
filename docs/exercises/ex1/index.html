Day 1 In-class Exercise

info

This is an in-class exercise. An exercise page like this one will contain a brief description but is intended to be supplemented by discussion during our meeting time. Complete the exercise to the best of your ability. Feel free to talk with other students as you work, and do not be afraid to ask questions if you get stuck. Aim to complete as much as possible during our meeting, and try to finish on your own before our next class meeting. Remember to submit on Gradescope once or more, checking the autograder results. For credit, you must submit by the end of the semester.

Learning Objectives#

Objectives

This exercise should help you gain familiarity with

  • Linux file system and commands
  • The basic structure of a C program
  • C integer variables
  • Basic text input and output in C
  • Arithmetic with integer values in C

Part 1 - Linux bootcamp#

Introduction#

Throughout this course, you will be working on a Linux system (CS Ugrad machine) remotely. Instead of programming with a graphical user interface (GUI) on your local PC, you will be coding via a terminal (a command-line interface) remotely. In this part, you will get yourself familiar with basic Linux file system and commands. The goal here is to familiar yourself with how to create/delete/change/move/copy folders/files in a Linux terminal.

If you are a Linux/Mac user, you may skip this part if you are already familiar with the Unix file system and commands; however, we encourage you to go through this bootcamp as a refresher of your Linux commands.

Linux terminal#

The first thing you need to work on this bootcamp is a Linux terminal. If you have a Linux/Max machine, feel free to use your own terminal. Otherwise, you may use this javascript emulated terminal.

For windows users

Windows now provide a Linux sub-system. You can run a Linux terminal natively on a Windows 10 machine. If you are interested in it, check out the info here for instructions.

Linux bootcamp#

Once you have your terminal ready, follow and complete this bootcamp.

Checkpoints#

After you have finished with the bootcamp, check if you can answer the below questions:

  • How to navigate to my home directory?
  • How to create create a directory and a file inside that directory?
  • How to move the file inside a directory out to the parent directory?
  • How to delete the entire directory including the sub-directories and files inside?
  • If you don't know how to use a Linux command, which command you can use to check the manual page?
  • What is a tab completion?

Part 2 - Hello world program in C#

Introduction#

In this part, you are going to see a "Hello world" program in C and get familiar with how a C program looks. Try to compare it with the "Hello world" program that you have seen in other programming languages.

Instructions#

  1. Start by going to the following website in a web browser: https://www.onlinegdb.com/online_c_compiler
  2. This website is a simple online IDE for C programs. You will see a "hello, world" program in the text editor, in a source file called main.c.
  3. Click the green "â–¶ Run" button just above the text editor. The program will be compiled, and then you will see the output Hello World in the console window below the text editor!
Note

We're using this online IDE for this exercise in order to make sure that everyone can start working on C programs right away, even if they don't yet have an account on the CS ugrad machines. All subsequent exercises (and assignments) should be done and well tested on the ugrad machines.

Checkpoints#

  1. Can we compile an executable without the main function in C?
  2. What is the return type of the main function?
  3. What does #include mean?

Part 3 - Basic text I/O in C#

Introduction#

stdio.h is the header file that provides input/output functions in C. In this part, you are going to modify the "Hello world" program to read in an integer value and print it out on the screen.

Instructions#

  1. To get started, change the program so that it looks like this:

    #include <stdio.h>
    int main(void) {
    int value;
    printf("enter an integer: ");
    scanf("%d", &value);
    printf("you entered: %d\n", value);
    return 0;
    }

  2. Click the green "â–¶ Run" button. The program should compile, and then the program will start running.

  3. You can use the console window (below the text editor) to interact with the program. Enter the value "42", and press "Enter". You should see the output you entered: 42 in the console window.

Checkpoints#

You have written a C program that reads user input and executed it interactively. Congratulations!

  1. Have you noticed the difference when you pass value in printf and in scanf?
  2. What does %d appearing in the format string mean?
  3. Can you modify it to scan and print a float?

Part 4 - Arithmetic operators and precedence#

Introduction#

We have learned operators and precedence in today's lecture. Let us practice it a little bit and examine the precedence properties of operators.

Instructions#

Your main task for this part is to write a program which

  • prompts the user to enter three integer values, denoted by a, b and c, and then
  • prints out the values of these expressions in the format specified below
    • a + b + c
    • a + b - c
    • a + b * c
    • a + b / c
    • a + b % c

Here is a sample run of the program, with bold text used to indicate the input entered by the user:

Enter two integers: 1 11 3
1 + 11 + 3 = 15
1 + 11 - 3 = 9
1 + 11 * 3 = 34
1 + 11 / 3 = 4
1 + 11 % 3 = 3
Note on scanf

Note that when asked to read multiple values, the scanf function doesn't care whether you enter the values on the same line, or on different lines. In the example session above, all values were entered on the same line.

Note on printf

One slightly tricky issue in producing the correct output is printing a literal % character using printf. In a printf format string, the % character usually is the start of a format specifier. However, you can have printf generate a literal % character by specifying two % characters in the format string. I.e., the code

printf("%%");

will print a single % character to the standard output.

Checkpoints#

  1. Is multiplication, division, modulus computed before addition, subtraction as you would expect?
  2. How does the integer division behave in C? Does it give you a float value?
  3. If you change the type from int to float, can you still do a + b % c?

Part 5 - Gradescope submission#

Copy and paste your code in Part 4 into a plain text file on your local computer called ex1.c. Make sure this is saved as a text file (not a special format like .docx) and does not also have a ".txt" extension. Submit it as Day 1 In-class Exercise on Gradescope (available at 10 am on June 7th). Make sure your program passes all the autograder tests in order to get full credit. It is due before the next class. However, late submission is allowed until our last class. Even so, we highly recommend you to keep up the schedule, finish each in-class exercise, and submit it on Gradescope before the next class.

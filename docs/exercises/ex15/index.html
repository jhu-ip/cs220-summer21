Day 15 In-class Exercise

info

This is an in-class exercise. An exercise page like this one will contain a brief description but is intended to be supplemented by discussion during our meeting time. Complete the exercise to the best of your ability. Feel free to talk with other students as you work, and do not be afraid to ask questions if you get stuck. Aim to complete as much as possible during our meeting, and try to finish on your own before our next class meeting. Remember to submit on Gradescope once or more, checking the autograder results. For credit, you must submit by the end of the semester, passing the auto-grader tests.

Learning Objectives#

Objectives

This exercise will help you gain familiarity with:

  • C++ classes, fields & member functions
  • Operator overloading
  • The friend keyword
  • Linked lists in C++
  • Rule of three
  • Writing template classes

Part 0 - Get the starter codes#

Get the starter codes from cs220-public, copy them over to cs220 (remember to create a new directory ex15 for that), and create an initial commit with the copied starter codes. You should see the following files: complex.[cpp|h], int_set.[cpp|h], int_node.h, main[x].cpp, and Makefile.

tip

If you don't remember how to get the starter codes, go back and check previous exercises for instructions.

Part 1 - A complex number class#

Introduction#

In this part, you will finish the implementation of a complex number class and practice C++ operator overloading.

Instructions#

  1. Read the instructional comments in complex.[cpp|h] and main1.cpp. It shows an example of how to use the given code for the Complex class. You can compile and run the code using the commands make main1 and ./main1.

  2. The first task is to overload and implement the operator<< function for printing out a complex number. You will also have to add the appropriate function declaration in complex.h. After your implementation, you can uncomment the related parts in main1.c, compile and test your implementation. (Note: Remember to use the friend keyword.)

  3. Follow the instructions in the comments in complex.[cpp|h] to write the missing functions for which we have provided declarations in the complex.h file. You should not change anything in the complex.h file for this part. Try not to reinvent the wheel, but take advantage of code reuse where possible. After your implementation, you can uncomment the related parts in main1.c, compile and test your implementation.

  4. Note that multiplying a float with a Complex object will not work as we have not overloaded the operator* for the float type. This can be done by adding a non-member function to allow a float to be multiplied with a Complex object. You will have to add the appropriate function declaration in complex.h and implement this function in complex.cpp. After your implementation, you can uncomment the related parts in main1.c, compile and test your implementation.

Checkpoints#

  1. What is operator overloading in C++?
  2. What is a friend function?
  3. When should we use the keyword friend to grant access to other classes? When should we make the member fields public instead?

Part 2 - A C++ linked-list class#

Introduction#

In this part, you will follow the instructions in the source code comments and finish the implementation of a linked list class in C++. You will practice implementing a C++ class, overloading operators, writing friend functions, and applying the rule of 3.

Instructions#

  1. Read main2.cpp. It shows an example of how to use the given code for the int_set class. You can compile and run the code using the commands make main2 and ./main2. Some of the statements in main2.cpp will not compile yet because of missing functions in int_set.cpp. You can comment them out to check that the supplied code is working correctly.

  2. Follow the instructional comments in int_set.cpp to write the operator<<, destructor, copy constructor, operator+=, and operator=, which we have prototyped in the int_set.h header. You should not change anything in the int_set.h file. Do not reinvent the wheel when implementing these functions! That is, consider how code reuse can make your job easier. You can test your implementation with main2, but will need to gradually uncomment some sections as you implement the above functions.

  3. Run valgrind on main2.cpp by calling valgrind --leak-check=full ./main2. Fix any memory leaks that it finds. Finally, add a new line in main2.cpp which assigns an existing non-empty set to itself, e.g. s = s; Investigate what occurs using valgrind! See if you are able to eliminate any problems that occur.

Checkpoints#

  1. What is the rule of 3? Which functions do you need to customize?
  2. Look it up online: In C++11, what is the rule of 5?
  3. What is the difference between the copy constructor and the assignment operator?

Part 3 - A C++ templated linked-list class#

Introduction#

In this part, you will transform the linked list class you just implemented to make it templated.

Instructions#

  1. Read main3.cpp. It shows an example of how to use a new template class named my_set. Before you can compile and run the code using commands make main3 and ./main3, you will need to finish the missing part in my_set.inc.

  2. Parts of the int_set have been transformed to the templated version my_set. Refer to your finished implementation above and complete the missing parts.

  3. The templated version should work with the templated my_node with any base type, so my_set can be used with any base type that has the operations == and < defined for it. (We must have these two defined, since we use them to maintain our sorted linked list, and to be able to avoid duplicates in our sets.)

  4. You can test your implementation with ./main3.

tip

1) Don't use my_set<T> inside the class definition for my_set (.h) file. You need it at the top of the class, but inside the class, use my_set instead.

2) Since operator<< is a friend rather than a member of the class, it will need a different template parameter (e.g., in my_set.h):

template<typename U>
friend std::ostream& operator<<(std::ostream& os, const my_set<U>& s);

3) In my_set.inc, use my_set<T>:: as the prefix for class method definitions.

  1. Run valgrind on main3.cpp. Fix any memory leaks that it finds.

Checkpoints#

  1. What is a template parameter? How is it used in a templated class / function?
  2. Why do we need another template parameter for the friend function?
  3. Where have you included the .inc file? Why can't we separate declaration and definition for a templated class's functions?

Part 4 - Gradescope submission#

Create a zip file called ex15.zip, which contains complex.[cpp|h], int_set.[cpp|h], int_node.h, my_set.[h|inc], and my_node.h with your implementation (you don't need to submit the main[x].cpp files and the Makefile. The autograder has its own Makefile and main.cpp for testing). Submit the zip file as Day 15 In-class Exercise on Gradescope. Make sure your program passes all the autograder tests in order to get full credit. It is due before the next class. However, late submission is allowed until our last class. Even so, we highly recommend you to keep up the schedule, finish each in-class exercise, and submit it on Gradescope before the next class.
